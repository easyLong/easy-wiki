# AI Short Drama Domain Framework

## Core Thesis

AI short drama should be produced as a controlled narrative system, not as disconnected AI-generated clips. The durable unit is the [[shot-unit]], and the durable control document is the [[continuity-bible]].

## Production Stack

```text
story force
  -> scene intention
  -> shot design
  -> visual continuity
  -> prompt execution
  -> edit rhythm
  -> sound continuity
  -> platform packaging
```

## Quality Dimensions

| Dimension | Owner | Main Question |
|---|---|---|
| Story force | [[script-expert]] | Why keep watching? |
| Scene intention | [[director-expert]] | What should the audience feel or understand? |
| Visual clarity | [[storyboard-expert]] | What should the viewer notice? |
| Image language | [[cinematography-expert]] | Does the image communicate the beat? |
| Generation control | [[prompt-expert]] | Can the model execute the shot repeatably? |
| Continuity | [[continuity-expert]] | Does the world remain stable? |
| Rhythm | [[editing-expert]] | Does every cut move attention? |
| Sound | [[sound-expert]] | Does audio clarify and connect the experience? |

## Default Decision Hierarchy

When tradeoffs appear, use this order:

1. Story and emotion.
2. Audience clarity.
3. Edit rhythm.
4. Character and world continuity.
5. Visual polish.
6. Tool convenience.
